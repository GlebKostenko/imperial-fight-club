# ИМПЕРИАЛ / APEX Fight Club — FastAPI + MongoDB

Сайт бойцовского зала с адаптированной мобильной главной страницей, отдельными страницами для подробной информации и админ-панелью.

## Что изменено

- Backend переведён с Node.js/Express на **Python + FastAPI**.
- Сохранены прежние API-маршруты `/api/*`, поэтому админ-панель продолжает работать с теми же сущностями.
- Главная страница переработана под мобильные устройства: на ней оставлены только ключевые блоки — зал, направления, тренеры, цены и запись.
- Детальная информация вынесена в отдельные страницы: `Расписание`, `Тренеры`, `Цены`, `Зал`, `Контакты`.
- Навигация работает через меню и CTA-кнопки на главной. Внутренние страницы доступны по прямым URL: `/schedule`, `/trainers`, `/pricing`, `/gallery`, `/contact`.

## Стек

- **Backend:** Python 3.12, FastAPI, Uvicorn, Motor
- **Database:** MongoDB
- **Frontend:** HTML/CSS/JavaScript
- **Proxy/Deploy:** Docker Compose + Nginx

## Быстрый старт

```bash
docker-compose build --no-cache
docker-compose up -d
```

Seed-скрипт запускается автоматически внутри backend-контейнера перед стартом FastAPI. Он заполняет демо-направления, тренеров, цены, галерею и создаёт администратора, если их ещё нет.

## Доступ

| Сервис | URL |
|---|---|
| Сайт | http://localhost |
| Админ-панель | http://localhost/admin |
| API / Swagger | http://localhost/docs |

Данные для входа в админку по умолчанию:

- Логин: `admin`
- Пароль: `password`

После запуска смените пароль в админ-панели.

## Основные команды

```bash
# Запуск
docker-compose build --no-cache
docker-compose up -d

# Логи backend
docker-compose logs -f server

# Повторное ручное заполнение/обновление демо-данных при необходимости
docker-compose exec server python seed.py

# Остановка
docker-compose down

# Остановка с удалением данных MongoDB
docker-compose down -v
```

## Фото на главной странице

Фотографии в первом split-блоке главной страницы меняются через админ-панель:

1. Откройте `http://localhost/admin`.
2. Перейдите во вкладку **Галерея**.
3. Нажмите **Добавить фото**.
4. Для левой фотографии выберите категорию **Главная: фото ринга слева**.
5. Для правой фотографии выберите категорию **Главная: борцовский зал справа**.
6. Загрузите изображение и сохраните.

Если фото не загружены, сайт использует встроенные изображения-заглушки из `public/assets`.

## API

Backend сохраняет совместимость с текущим фронтендом и админкой:

- `POST /api/auth/login`
- `GET /api/auth/verify`
- `PUT /api/auth/password`
- `GET/POST/PUT/DELETE /api/trainers`
- `GET/POST/PUT/DELETE /api/directions`
- `GET/POST/PUT/DELETE /api/pricing`
- `GET/POST/PUT/DELETE /api/gallery`
- `GET/POST/PUT/DELETE /api/contacts`
- `POST /api/upload/{type}`

## Примечание по данным

Коллекции MongoDB оставлены совместимыми с прежним Mongoose-проектом: `trainers`, `directions`, `pricings`, `galleries`, `contacts`, `admins`.


## Примечание по bcrypt

Backend использует `passlib==1.7.4` с закреплённой версией `bcrypt==4.0.1`. Это предотвращает ошибку `module bcrypt has no attribute __about__` и сбой seed-скрипта в Python 3.12.

## Тренеры и специализации

Во вкладке **Тренеры** поле **Специализации** теперь поддерживает несколько значений через запятую, например:

```text
Бокс, кикбоксинг
```

На публичной странице тренера отображаются только имя, фото, стаж и специализации. Пометки уровней групп (`Дети`, `Взрослые`, `Новички`, `Продвинутые`) оставлены только для занятий во вкладке **Расписание**.

## Замена фотографий в первом блоке главной

Во вкладке **Галерея** добавление фото для категорий **Главная: левая фотография ринга** и **Главная: правая фотография борцовского зала** теперь работает как замена текущего изображения для этой позиции. То есть не нужно удалять старое фото: выберите нужную категорию, загрузите новое изображение и нажмите **Сохранить**.

## Изменения в расписании и тренерах

- В занятиях расписания больше нет отдельных полей «цена за занятие» и «цена за месяц». Цены редактируются только во вкладке «Тарифы».
- Для каждого слота занятия можно указать:
  - день,
  - время,
  - тренера,
  - уровень группы,
  - кто занимается: Мужчины / Женщины / Все.
- У тренеров поле специализаций заменено на множественный выбор направлений. Эти направления одновременно используются как фильтры и как подпись на карточке тренера.
- В seed-данных используются направления: Вольная борьба, Бокс, Кикбоксинг, Дзюдо и Самбо.

## Обновление формы создания занятий

- В админке во вкладке «Расписание» дни недели выбираются галочками из выпадающего списка.
- Один слот может относиться сразу к нескольким дням, например: Пн, Ср, Пт.
- Время задаётся двумя полями: начало и конец занятия.
- Все варианты времени идут с шагом 30 минут: 06:00, 06:30, 07:00 и так далее.
- При сохранении время записывается в формате `HH:MM — HH:MM`, поэтому публичная таблица расписания продолжает работать без дополнительных действий.

## Обновления админки

- Кнопка **Связаться** в шапке сайта ведёт в контактный блок внизу главной страницы и остаётся видимой на телефоне.
- В админке во вкладке **Расписание** доступны две операции:
  - **Добавить направление** — создать новое направление без слотов расписания;
  - **Добавить занятие** — создать направление/занятие со слотами расписания.
- В слотах расписания тренер выбирается из выпадающего списка существующих тренеров.
- Время занятий выбирается промежутком с шагом 30 минут.
- Мобильная версия расписания уплотнена: таблица стала меньше и прокручивается по горизонтали.


## Обновления админки и расписания

- Логотип в админ-панели и пункт «На сайт» в мобильном нижнем меню ведут на главную страницу.
- Форма занятия адаптирована под мобильный экран: дни, время, тренер, уровень, возраст и пол размещены без горизонтального скролла.
- В занятии добавлен параметр «Возраст»: Дети, Подростки, Взрослые, Все.
- Фильтры тренеров на сайте строятся из направлений, добавленных в админке.
- Мобильная страница расписания показывает занятия карточками по дням недели, чтобы день всегда был виден при прокрутке.
- Направления в расписании визуально разделены на ударные виды спорта и борьбу.

## Обновление админки: направления и занятия

- Направления теперь редактируются в отдельной вкладке **Направления**.
- У направления можно указать `slug`, название, описание и цвет. Цвет используется в карточках направления и в расписании.
- Вкладка **Расписание** теперь создаёт занятия только для уже существующих направлений: в форме занятия выберите `Slug (URL)` из выпадающего списка, а название подтянется автоматически.
- Расписание группируется по времени начала занятия. Если два занятия начинаются в один день и в одно время, но заканчиваются в разное время, они отображаются в одной строке; полный промежуток времени показан внутри карточки занятия.

## Последние изменения

- В админ-панели вкладка «Направления» перенесена вниз рядом с «Настройки».
- В «Настройках» добавлено редактирование текста главной страницы и контактного блока.
- Контактные данные теперь можно добавлять и удалять интерактивно: Telegram, Instagram, Max, WhatsApp, телефон, адрес, email и другие ссылки.
- Публичная главная страница и страница контактов берут текст и контакты из `/api/settings`.
- Расписание переработано в часовую сетку: слева фиксированные часовые метки, карточки занятий растягиваются по длительности занятия. Например, слот 17:00–18:30 занимает один полный час и половину следующего.
- Если занятия пересекаются по времени в один день, они отображаются в одной временной сетке рядом, а не создают новые строки.

## Обновление: футер, соцсети и бюджетные места

- В нижней панели сайта блок «Контакты» показывает только адрес, телефон и режим работы.
- Социальные сети вынесены в отдельный блок «Соцсети» и редактируются в админке во вкладке «Настройки».
- Страница `/budget` («Бюджетные места») редактируется в отдельной вкладке админ-панели «Бюджетные места»: можно менять заголовок, вводный текст, фотографию и условия с подкатегориями.
- Для направлений в админке доступен выбор только из двух цветов: жёлтый и голубой.

## Управление блоками главной страницы

В админ-панели теперь можно отдельно выбирать, какие элементы показывать на главной странице:

- **Тренеры**: в карточке тренера есть чекбокс «На главной» и стрелки для изменения порядка.
- **Направления**: во вкладке «Направления» есть чекбокс «На главной» и стрелки вверх/вниз.
- **Цены**: в карточке тарифа есть чекбокс «На главной» и стрелки для изменения порядка.

Если ни один элемент в категории не отмечен, сайт автоматически показывает первые активные элементы по порядку, чтобы блок на главной не был пустым.


## Обновление карточек тренеров

На главной странице у тренера отображаются направление, опыт и все достижения, указанные в карточке тренера. Отдельный выбор достижений для главной больше не используется.

## HTTPS на Ubuntu-сервере

В проект добавлена production-конфигурация для HTTPS через nginx и Let's Encrypt.

1. Укажите DNS `A`-записи домена на IP сервера:

```text
@    A    IP_СЕРВЕРА
www  A    IP_СЕРВЕРА
```

2. На сервере откройте порты `80` и `443`.

3. Создайте файл `.env.production`:

```bash
cp .env.production.example .env.production
nano .env.production
```

Укажите ваш домен, www-домен, email и `JWT_SECRET`.

4. Получите сертификат и запустите HTTPS-конфигурацию:

```bash
./issue-cert.sh
```

После выпуска сертификата проект запускается так:

```bash
docker compose -f docker-compose.https.yml --env-file .env.production up -d --build
```

Для продления сертификатов можно выполнить:

```bash
docker compose -f docker-compose.https.yml --env-file .env.production run --rm certbot renew
docker compose -f docker-compose.https.yml --env-file .env.production exec nginx nginx -s reload
```

MongoDB и FastAPI больше не публикуются наружу отдельными портами. Снаружи доступны только nginx-порты `80` и `443`.

## Загрузка фотографий

В админке при загрузке фото появляется окно предпросмотра. Жёлтая рамка показывает область, которая попадёт на сайт. Фотографию можно двигать мышью или пальцем, а масштаб менять ползунком. Для разных мест сайта используется разное соотношение кадра: карточка тренера, фото главной, галерея или страница «Бюджетные места».

## Развёртывание на Ubuntu-сервере с HTTPS

В проект добавлен скрипт `deploy-server.sh`, который выполняет основные команды для production-запуска:

- устанавливает Docker и Docker Compose plugin, если их нет;
- включает автозапуск Docker после перезагрузки сервера;
- создаёт `.env.production`;
- выпускает HTTPS-сертификат Let’s Encrypt;
- запускает контейнеры через `docker-compose.https.yml`;
- добавляет cron-задачу для автоматического продления сертификата.

Перед запуском убедитесь, что DNS A-записи домена указывают на IP сервера, а порты `80` и `443` открыты.

Первичное развёртывание:

```bash
chmod +x deploy-server.sh issue-cert.sh
./deploy-server.sh setup imperial-fight.ru admin@imperial-fight.ru www.imperial-fight.ru
```

Обычный запуск или обновление после изменения кода:

```bash
./deploy-server.sh start
```

Перезапуск без пересборки:

```bash
./deploy-server.sh restart
```

Просмотр логов:

```bash
./deploy-server.sh logs
```

Проверка контейнеров:

```bash
./deploy-server.sh status
```

Ручное продление сертификата:

```bash
./deploy-server.sh renew-cert
```

Остановка проекта:

```bash
./deploy-server.sh stop
```

MongoDB и FastAPI в production-конфигурации не публикуются наружу. Внешние порты открыты только у nginx: `80` и `443`.
